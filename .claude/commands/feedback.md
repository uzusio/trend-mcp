# ニュース選択のフィードバック

直前の `/post-trend` の結果に対してフィードバックを行い、傾向ファイルを更新します。

## 手順

1. `get_post_history` で直前に投稿したニュースを確認

2. ユーザーに以下を確認:
   - 直前に選ばれたニュースは良かったか？（良い / 悪い）
   - その理由は？（任意）

3. `record_feedback` でフィードバックを記録（統計は自動でカウントされる）
   - rating: "good" または "bad"
   - reason: ユーザーの理由（任意）
   - newsTitle: 対象のニュースタイトル（任意）

4. `config/preferences.md` を読み込む

5. フィードバック内容に基づいて傾向ファイルを更新:
   - **良い場合**: 「興味のある傾向」セクションに、なぜ良かったかの要素を追加
   - **悪い場合**: 「興味のない傾向」セクションに、なぜ悪かったかの要素を追加
   - **注意**: 傾向は抽象化して記録する（具体的なニュースタイトルは書かない）

6. 更新した傾向ファイルを保存

## 例

ユーザー: 「悪い。スポーツの結果には興味ない」

→ `record_feedback` で rating:"bad", reason:"スポーツの結果には興味ない" を記録
→ 「興味のない傾向」に「スポーツの試合結果」を追加

ユーザー: 「良い！新作ゲームの情報は嬉しい」

→ `record_feedback` で rating:"good", reason:"新作ゲームの情報は嬉しい" を記録
→ 「興味のある傾向」に「ゲームの新作発表・続報」を追加
